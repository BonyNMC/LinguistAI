## 📅 Change Log & Key Decisions
- **2026-03-23**: Confirmed isolated project "LinguistAI". Multi-Provider Support added. Fixed 401 Unauthorized via internal JWT validation. Added `sessionStorage` persistence to Writing Space. Implemented explicit Recall Prompt Strategy.
- **2026-03-24**: **Phase 11**: Transitioned to BYOK and commercial access model. Added `account_status` and `role`. Edge Function gating.
- **2026-03-24**: **Phase 12**: Web Speech API TTS. Mobile Responsive updates. Added SM-2 focus word selection (top 5) for Active Recall Report and Native Rewrite modeling. HTML sanitization.
- **2026-03-25**: **Phase 13**: UX & Admin Fixes. Fixed Auth white screen. Added admin realtime notifications for pending registrations. Fixed Admin Dashboard layout offset. Added Dark/Light mode toggle. Added WaitingRoom CSS.
- **2026-03-25**: **Phase 14**: Writing Data Export. Added `ExportPanel.jsx`. Period filter, Token size badge, Markdown file download. Allowed pure database saving without API key via **Save** button. Added Custom date range picker and Last N entries filters.
- **2026-04-06**: **Phase 15**: 5 major new features — (1) **AI Conversation Mode** (`/conversation` page, `conversation_sessions` DB table, `chat-message` + `analyze-conversation` Edge Functions). (2) **Shadowing / Micro-Speech** (Web Speech API in Review result phase). (3) **Linked Story Challenges** (`generate-challenge` updated with `story_mode`/`story_context` params). (4) **CEFR Implicit Profiling** (3 new columns on `user_profiles`, Weighted Moving Average detection in `analyze-writing` + `analyze-conversation`, i+1 hypothesis in prompts, CEFR card in Settings). (5) **Leaderboard** (`/leaderboard` page, `leaderboard` DB view, `user_public_profiles` table, display_name opt-in/out in Settings).

- **2026-04-07**: **Phase 16**: Active Recall UX upgrade. (1) **Conversation Mode is now the default landing page** (`/conversation` replaces `/writing` as the root redirect). NavBar order updated: Conversation first. (2) **Daily Vocab Missions** (`DailyVocabMissions.jsx` component + `generate-daily-mission` Edge Function): Before writing or chatting, users see their 3-5 weakest SM-2 words and an AI-generated topic/mission that naturally invites using those words. Mission is cached in `sessionStorage` to avoid redundant API calls across tab switches.
- **2026-04-07**: **Phase 16 (fix)**: Active Recall closure loop. (1) **`creditVocabUsage()` helper** added to `analyze-conversation` (v2) and `analyze-writing` (v15): after analysis, whole-word matches against the user's Study List award +10 mastery each and update SM-2 schedule — so using words in Writing/Conversation is now rewarded, not just Review. (2) **Diverse vocab type suggestions**: `analyze-writing` and `analyze-conversation` prompts now enforce at least 1 `phrasal_verb`, 1 `linking_word`, and 1 `idiom` per suggestion batch (not just `vocab`). (3) **"🏅 Mastery Credited!" banner** shown in `AnalysisPanel` (ConversationMode) and `AnalysisResult` (WritingSpace) whenever words are credited.
- **2026-04-07**: **Phase 17**: Session History + Stats + Leaderboard Streak. (1) **`/history`** page (`History.jsx`): merged writing + conversation sessions grouped by day, client-side search (across title/raw content/transcript), pagination (20/page), expandable cards showing full writing text or conversation transcript + analysis summary. (2) **`/stats`** page (`Stats.jsx`): 4 summary cards (streak, words, mastery, sessions), Activity Heatmap (6 months, CSS grid), Daily Practice line chart (30 days, pure SVG). (3) **Leaderboard**: added `🔥 Best Streak` sort tab + streak column in table. (4) **Streak tracking**: `updateStreak()` helper in `analyze-writing` (v16) + `analyze-conversation` (v3), updates `current_streak`, `best_streak`, `last_activity_date` in `user_profiles`.
- **2026-04-08**: **Phase 18**: Metalinguistic Feedback & Contextual Tone Analysis. (1) **Scenario Context** parameter added to WritingSpace and ConversationMode. (2) Edge functions `analyze-writing` (v17) and `analyze-conversation` (v4) updated to interpret context. (3) Added `tone_evaluation` (Tone Detected, Appropriateness, Suggestion) to analysis results. (4) Extracted `explanation` field to `error_highlights` to deliver explicit grammar rules alongside corrections.

- **2026-04-08**: **Phase 19**: SRS Lifecycle & Maintenance Review. (1) **Fixed SRS lifecycle bug**: `Review.jsx` was only querying `status = 'learning'`, stranding all `reviewing` words. Fixed to query `['learning', 'reviewing']`. (2) **Correct mastery thresholds**: `mastered` now requires mastery=100 (not 80). mastery 80-99 = `reviewing` (consolidation). mastery <10 = `learning` (lapsed). (3) **Fixed `linking_verb` → `linking_word`** word type taxonomy in `StudyList.jsx` and all prompts. (4) **Data integrity migration**: corrected pre-existing status/mastery mismatches in DB (old `mastered` rows with mastery<80 reset to `learning`). (5) **StudyList UX**: added 🔄 Refresh button, `useLocation`-based auto-refetch when navigating to page, `reviewing` added to status filter dropdown and `STATUS_COLORS`. Subtitle now shows reviewing+mastered counts. (6) **Leaderboard formula panel**: expanded from 1-line formula to 4-card breakdown showing how each activity (Writing/Review/Mastery/Streak) earns points. (7) **Maintenance Review**: mastered words re-enter Review queue every 90 days. Pass → stay mastered (+90 days). Fail → mastery=70, status=reviewing. Max 3/session. Yellow 🔧 badge in UI. No automatic score decay (evidence-based: decay only on demonstrated forgetting). Logic in `calcMaintenanceResult()` separate from SM-2 `calcNextReview()`.

- **2026-04-10**: **Phase 20 — Bugfixes**: (1) **`vocab_master_type_check` constraint fixed**: DB constraint was still using `linking_verb` (old wrong name) instead of `linking_word`. Migration applied to drop and recreate constraint with correct value, plus UPDATE to fix any pre-existing rows. (2) **Daily Mission topic bias fixed**: `generate-challenge` and `generate-daily-mission` fallback was `'General professional English'` causing work-focused content even for non-professional users. Changed to `'everyday life, hobbies, travel, and personal experiences'`. Mission prompt strengthened with CRITICAL RULES section explicitly forbidding defaulting to office/work scenarios. (3) **Settings save now clears Daily Mission cache**: `sessionStorage.removeItem('linguist_daily_mission')` called on successful profile save so new focus_topic takes effect immediately on next page visit.

- **2026-04-10**: **Phase 21 — Tier 1 & Tier 2 Pedagogical Roadmap**: (1) **Graded Reading** (`/reading` page): `generate-reading` Edge Function generates i+1 CEFR passages (180–240 words) embedding user's study words + 3 comprehension questions (factual/vocab-in-context/inference). `reading_sessions` table + RLS. Passage highlights study words with `.mark-recall` class. Recent session history panel. (2) **Error Pattern Intelligence** (Stats page): aggregates `error_highlights[]` from last 30 writings + 30 conversations, renders bar chart by type, lists top 5 recurring `original → corrected` pairs. (3) **Contextual Cloze Review** (Review page, v1): `generate-cloze` Edge Function + Cloze tab in Review — single-blank cloze per current word (SUPERSEDED by Phase 22). (4) **Enhanced Shadowing**: word-level diff (green=hit, red=miss), IPA phonetic from Dictionary API, max 3 attempts. (5) **Writing Genre Scaffolding**: Genre selector (6 genres) in WritingSpace, structure outlines, clickable useful phrases, `genre` param sent to `analyze-writing`.

- **2026-04-11**: **Phase 22 — Cloze Redesign + Grammar Practice + Leaderboard Fix** (commit b2ba223): (1) **Leaderboard bug fixed**: `handleSubmitCloze` was sync/fire-and-forget; changed to `async` with `await` per DB update — leaderboard now reflects cloze scores immediately. (2) **Cloze redesigned** (`generate-cloze` v2): now selects 3–5 due words autonomously (no current word card dependency), generates ONE connected passage with numbered blanks `[1]`…`[N]`. Word bank shown as reference — user MUST type manually (deliberate: builds orthographic memory; no dropdown/click-to-assign). `<ClozePassage>` inline component renders inputs inside text flow. Per-blank SRS grading with `await`. (3) **Grammar Practice** (`generate-grammar-exercise` Edge Function NEW): reads last 60 sessions' grammar errors, LLM identifies primary weakness, generates 5 targeted MCQ drills. Review page gets 4th mode "📏 Grammar". Stats page gets "🎯 Practice" button next to grammar error bar → `/review?grammar=grammar`. Grammar is a separate skill track (does NOT touch SRS/`user_vocab_progress`).

- **2026-04-11**: **Phase 23 — Professional Audit & Database Optimization**: Full-stack codebase audit conducted. (1) **RLS initplan fix** (migration `optimize_rls_initplan_and_add_indexes`): all 7 public tables' RLS policies updated from `auth.uid()` to `(select auth.uid())` — prevents per-row re-evaluation, significantly improves query performance at scale. (2) **Missing FK indexes added**: `idx_reading_sessions_user_id`, `idx_user_vocab_progress_vocab_id`, `idx_user_writings_user_id` — prevents sequential scans on JOIN/filter queries. (3) **Merged duplicate UPDATE policies** on `user_profiles`: "Users can update own profile" + "Admins can update all profiles" → single "Users or admins can update profiles" policy. (4) **Fixed `increment()` function** search_path security (`ALTER FUNCTION SET search_path = public`). (5) **Fixed Leaderboard date mutation bug** (`Leaderboard.jsx`): `now.setDate()` was mutating the original Date object → "This Week" filter could show incorrect date ranges. Fixed by creating a copy before mutation. (6) **Fixed sessionStorage data leak on logout** (`NavBar.jsx`): all `linguist_*` keys now cleared on sign-out, preventing user B from seeing user A's cached writing drafts, conversation history, and daily missions after switching accounts on same tab. (7) **`vocab_master` open INSERT/UPDATE policies confirmed as accepted risk** — intentional shared dictionary design; each user's progress is isolated in `user_vocab_progress` with proper RLS. (8) **Admin bypass of AccessGuard confirmed as intentional** — single admin (project owner) must always be able to access `/admin` even if accidentally self-suspended.
